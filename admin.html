إدارة الصلاحيات والاشتراكات

الأدوار Student, Teacher, Parent, Admin مع إمكانية ضبط الصلاحيات لكل دور في النسخة الخلفية لاحقًا.
الباقات مجاني محدود، اشتراك طالب، اشتراك معلم، اشتراك مدارس مع حدود رسائل وXP مختلفة.
الرصيد XP يُخصم محليًا مع كل رسالة داخل النموذج الأولي لإظهار الاستهلاك في لوحة الأدمن.

إدارة الذكاء الاصطناعي

المعادلة المعتمدة نموذج مفتوح المصدر + RAG للمنهج السعودي + LoRA/SFT للسلوك + تشغيل محلي عبر vLLM أو Ollama.
المعرفة مقابل السلوك الكتب والحلول والشروحات تبقى داخل طبقة المعرفة، بينما تنسيق الرد ونبرة ملم يحل تضبط عبر التدريب السلوكي.
سياسة الرد الإجابة النهائية، الشرح، الخطوات، الأخطاء الشائعة، والسؤال المشابه وفق Policy Layer خاصة بالمنصة.
طبقة التصنيف والمراجعة مصنف نية مستقل يحدد نوع الرسالة أولًا، ثم طبقة بحث ويب تجمع مراجع وتبني قرار ترجيح قبل إرسال الجواب للطالب.

مخازن البيانات المطلوبة

Knowledge Data كتب المنهج، الحلول النموذجية، الشروحات، الأسئلة السابقة، والتعاريف.
Behavior Data عينات توضح كيف يشرح ملم يحل، متى يرفض الجواب المباشر، وكيف يرتب الرد حسب المادة والمرحلة.
Evaluation Data مجموعة تقييم داخلية لقياس الدقة، الالتزام بالمنهج، جودة العربية، وصحة خطوات الحل.

خطة التشغيل خلال 30 يوم

الأسبوع 1 اختيار النموذج، تجهيز السيرفر، وبناء ingestion للمنهج مع embeddings وretrieval.
الأسبوع 2 بناء prompt موحد واختبار 100 سؤال وتجميع الفجوات.
الأسبوع 3 إعداد dataset سلوكي والبدء في SFT أو LoRA.
الأسبوع 4 تقييم، دمج adapter، وتشغيل داخلي مع A/B test.
